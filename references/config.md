# Protonskill Configuration

The helper reads account settings from:

```text
~/.openclaw/state/protonskill/config.json
```

Override this path with `EMAIL_MANAGER_CONFIG=/path/to/config.json`.
The CLI also accepts `--config /path/to/config.json` before the subcommand.

## Create A Sample Config

Recommended guided setup:

```bash
python3 ~/.openclaw/skills/protonskill/scripts/configure_wizard.py
```

The wizard asks one question at a time and writes the JSON config without asking
for or storing the password.

Manual sample:

```bash
python3 ~/.openclaw/skills/protonskill/scripts/email_cli.py init-config --print
```

To write the sample file:

```bash
python3 ~/.openclaw/skills/protonskill/scripts/email_cli.py init-config
```

## Config Shape

Store hostnames and environment variable names in JSON. Do not store passwords in JSON.

```json
{
  "default_account": "proton",
  "accounts": {
    "proton": {
      "from": "Your Name <you@proton.me>",
      "imap": {
        "host": "127.0.0.1",
        "port": 1143,
        "ssl": false,
        "mailbox": "INBOX",
        "user_env": "EMAIL_PROTON_USER",
        "password_env": "EMAIL_PROTON_PASSWORD"
      },
      "smtp": {
        "host": "127.0.0.1",
        "port": 1025,
        "ssl": false,
        "starttls": false,
        "user_env": "EMAIL_PROTON_USER",
        "password_env": "EMAIL_PROTON_PASSWORD"
      }
    }
  }
}
```

Set credentials in the shell or OpenClaw secret environment, for example:

```bash
export EMAIL_PROTON_USER="bridge-generated-username"
export EMAIL_PROTON_PASSWORD="bridge-generated-password"
```

## Proton Mail Bridge

Proton Mail requires Proton Mail Bridge for local IMAP/SMTP access. The skill
does not connect directly to Proton Mail servers.

Default Bridge ports:

- IMAP host: `127.0.0.1`, port `1143`, SSL `false`
- SMTP host: `127.0.0.1`, port `1025`, SSL `false`, STARTTLS `false`

If Bridge shows different ports, use the ports displayed in Bridge settings.
Use the IMAP/SMTP credentials generated by Bridge, not the normal Proton account password.

## Other Providers

Gmail:

- IMAP host: `imap.gmail.com`, port `993`, SSL `true`
- SMTP host: `smtp.gmail.com`, port `465`, SSL `true`
- Use an app password for accounts with 2-Step Verification.

Outlook / Microsoft 365:

- IMAP host: `outlook.office365.com`, port `993`, SSL `true`
- SMTP host: `smtp.office365.com`, port `587`, SSL `false`, STARTTLS `true`
- Some tenants require OAuth and will reject password auth.

Generic IMAP/SMTP:

- Prefer IMAP over SSL on port `993`.
- Prefer SMTP SSL on port `465` or STARTTLS on port `587`.
- Confirm whether the SMTP username differs from the IMAP username.

## Validation

```bash
python3 ~/.openclaw/skills/protonskill/scripts/email_cli.py check-config
python3 ~/.openclaw/skills/protonskill/scripts/email_cli.py check-config --connect
```

`--connect` performs real IMAP/SMTP login checks. Use it only when the user expects a live account test.
